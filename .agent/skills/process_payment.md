# 功能引擎：金流付款處理 (Skills)

## 1. 基礎實作規劃

本技能模擬真實世界結帳時的支付網關 (Payment Gateway) 互動邏輯：

### 核心執行步驟

- **發票設定載入**：在進行結帳前，先接收並暫存顧客選擇的「發票載具條碼」或「統一編號」。
- **連線模擬**：啟動結帳後，使用 `setTimeout` 等待 2 秒模擬網路傳輸。
- **結果隨機分發**：透過亂數模擬不同金流回傳結果。

## 2. 進階實作規格 (AI好理解版)

### 金流狀態分配矩陣 (Probability Matrix)

| 隨機值區間 | 模擬結果 (Result) | 觸發後續機制 |
| :--- | :--- | :--- |
| `0.0 ~ 0.3` | **Timeout** (連線逾時) | 觸發 `payment_timeout` 規則，顯示重試按鈕 |
| `0.3 ~ 0.6` | **Insufficient_Funds** (餘額不足) | 觸發 `payment_timeout` 規則，顯示更換付款方式 |
| `0.6 ~ 1.0` | **Success** (結帳成功) | 清空購物車，轉入完成頁面 |
