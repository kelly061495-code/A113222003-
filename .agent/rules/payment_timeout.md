# 業務規則：付款失敗與超時機制

## 1. 基礎實作規劃

本規則確保顧客在付款失敗時，訂單不會直接消失，提供友善的補救流程：

- **異常偵測**：系統模擬金流連線時，可能發生「連線逾時」或「餘額不足」。
- **訂單保留**：當發生異常時，系統不應清空購物車，而是進入保留緩衝狀態。
- **緩衝時間**：提示顧客「系統已為您保留訂單 10 分鐘緩衝時間」。
- **補救措施**：顯示「重新付款」或「更改付款方式」的按鈕，引導顧客完成交易。

## 2. 進階實作規格 (AI好理解版)

### 付款狀態決策表 (Payment State Matrix)

| 金流回傳狀態 | 系統動作 | UI 顯示選項 |
| :--- | :--- | :--- |
| **Success** (成功) | 訂單成立，清空購物車 | 無，跳轉至完成頁面 |
| **Timeout** (逾時) | 保留購物車資料，倒數 10 分鐘 | 「重新付款」、「更改付款方式」 |
| **Insufficient_Funds** (餘額不足)| 保留購物車資料，倒數 10 分鐘 | 「更改付款方式」 |
