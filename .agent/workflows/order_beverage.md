# 操作流程：智慧點餐 SOP (含進階流程)

## 1. 基礎實作規劃 (人類好閱讀版)

本流程以狀態機方式描述 App 從啟動到結單的標準動線，支援「多杯點餐」與「異常處理」：

### 流程狀態定義

- **S0 (環境與狀態感知)**：App 啟動，取得氣溫決定首頁主打；取得門市狀態與原物料剩餘量。
- **S1 (需求選取與購物車)**：顧客選擇飲品、配料數量，點擊「加入購物車」。
- **S2 (迴圈確認)**：詢問顧客「是否繼續點餐？」。若「是」，折返 S1；若「否」，進入 S3。
- **S3 (購物車總覽與修改)**：展示完整訂單，顧客可刪除或修改。系統檢查 `delivery_threshold` 及原物料庫存。
- **S4 (會員與折扣計算)**：詢問是否登入會員。計算總額（含 95 折會員優惠與滿千折百）。
- **S5 (發票與結帳準備)**：詢問發票載具或統編設定。
- **S6 (付款處理與異常應對)**：進行金流連線。
  - 若成功：進入 S7。
  - 若失敗 (超時/餘額不足)：啟動保留緩衝時間（10 分鐘），提示顧客「重新付款」或「更換付款方式」，折返 S6。
- **S7 (自動通知)**：通過 `notification_consent` 檢查後，呼叫 `line_messaging_skill` 發送推播。
- **S8 (雲端存檔)**：呼叫 `db_storage_skill` 將訂單資料存入 Turso 云端。

## 2. 進階實作規格 (AI好理解版)

### 系統執行狀態表 (State Transitions)

| 階段序號 | 執行動作名稱 | 動作內容描述 | 關鍵產出數據 / 條件轉移 |
| :--- | :--- | :--- | :--- |
| **S0** | **Init_Context** | 獲取天氣、庫存、門市狀態 | 推薦飲品、可用庫存量 |
| **S1** | **Add_To_Cart** | 採集單品需求並暫存購物車 | Cart_Items Array |
| **S2** | **Order_Loop** | 詢問是否繼續點餐 | If True -> S1, Else -> S3 |
| **S3** | **Review_Cart** | 購物車總覽，可刪除單品 | 最終確定之 Cart_Items |
| **S4** | **Calc_Price** | 執行會員登入判斷與金額計算 | Net_Total, Discount_Sum |
| **S5** | **Invoice_Setup**| 發票載具與統編設定 | Invoice_Type, Carrier_ID |
| **S6** | **Process_Pay** | 模擬金流請求與錯誤處理 | If Success -> S7, If Fail -> Retry S6 |
| **S7** | **Push_Notify** | 檢查權限並發送 LINE 推播 | line_messaging_skill |
| **S8** | **Persistence** | 將結帳資料寫入 Turso 雲端 | db_storage_skill |
